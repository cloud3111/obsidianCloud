# 深度学习与智能体(Agent)技术
## 大语言模型(LLM)和视觉模型(VM)和智能体(AGENT)
- 大语言模型: 基于 Token 进行上下文概率预测与文本生成 充当智能体的“大脑”
- 视觉模型: 负责感知层 涵盖图像分类 语义分割 目标检测等任务 充当智能体的“眼睛”
- 智能体: 以 LLM 或 VM 为核心引擎 通过 Python 或 Java 编写控制逻辑 赋予模型调用工具、自主决策与长期记忆的能力 典型代表如基于 Spring AI 构建的自动化购票系统
## 计算机视觉模型训练精要
### 数据集构建
- 训练集格式: 图像与标签的映射 常用格式包括 VOC 格式与 YOLO 格式
- 数据索引: 通过脚本生成图像与标签的索引文件 严格划分训练集与验证集 比例通常为 8:2
- 图像标注: 通过人工或半自动工具标记目标特征 标注质量直接决定模型上限
- 当然也可以通过下载VOC12+07的数据集，但是如果要通过微调来指针具体的业务场景进行业务训练，像我们之前那个公司是做无人值守便利店，接入了视觉模型对具体的顾客拿烟行为进行判定，具体的训练数据也是外包出去通过人工标注来获得的，以此达到模型微调的目的，因为大模型很笨，所以只有通过大量的数据训练来进行进化，不单单是视觉模型，ai对话模型像ChatGPT，deepseek也是通过大量数据来训练的，为什么近些年ai好像不像前些年那样进步飞快了，原因很简单，训练数据都是基于人类本身的，现存的网络数据库的，当某一天模型的训练速度大于数据的生成速度时，自然而然就进步缓慢了，如果要解决这个问题，就要虚构一些不存在的数据，这就会导致另一个问题的发生: 幻觉！
### 模型训练
1. 参数
2. 是否使用CUDA
3. 选择主干网络
4. 输入图像大小(输入尺寸)
5. 每次训练数量BatchSize
6. Epoch循环次数
7. 是否使用权重文件pth(model_path)
8. 区分类个数(numble_classes) 重要
9. 工作线程数(workers)
### 模型验证阶段
- 数据集划分: 训练集  验证集  测试集
- 问：为什么要冻结训练和解冻训练呀？ 
- 答：这是迁移学习的思想，因为网络主干特征提取部分所提取到的特征是通用的，我们冻结起来训练可以加快训练效率，也可以防止权值被破坏。 在冻结阶段，模型的主干被冻结了，特征提取网络不发生改变。占用的显存较小，仅对网络进行微调。 在解冻阶段，模型的主干不被冻结了，特征提取网络会发生改变。占用的显存较大，网络所有的参数都会发生改变
### 指标
- 判断损失函数是否有收敛的趋势
- 判断是否过拟合(在训练集上表现良好，但在验证集没出现过的数据上准确率低)
- 判断精确率
- 判断召回率
### 微调
- 微调：像让学生复读，改变的是模型内部的权重（参数知识），适合改变语气或学习特定领域词汇
- RAG：像让学生开卷考试，改变的是外部输入的上下文（外部知识），适合处理频繁更新的数据（如最新的库存、价格）

### Prompt提示词
- 让一位说书人(角色)，用生动的语气(风格语气)，给孩子们(受众)讲个故事(指令)。故事的开头(上下文是…结局(目标)要感人。故事的结构(格式)要像这样(示例)，但不要(约束)出现暴力情节。请先构思情节(逐步思考)，写完后再想想怎么能更精彩(反思)
### Rag向量库
- RAG（检索增强生成）：为LLM注入知识与记忆
- 定位：解决LLM的“静态知识”和“幻觉”问题的核心技术。
- 核心流程：
	- 索引：将私有知识库（文档、手册、代码等）进行分块、向量化，并存入向量数据库。
	- 检索：根据用户查询，从向量库中快速检索出最相关的知识片段。
	- 增强：将检索到的片段作为上下文，与用户查询一同提交给LLM。
	- 生成：LLM基于提供的权威上下文生成更准确、可靠的答案
- 原则上可以利用所以现存的数据库, 包括关系型和非关系型数据库来做,  但是比较常用的比较适配的有PostgreSQL与PgVector, 在我自己的项目中使用的是ES作为向量数据库
