- String是一种包装类, 同样继承了Object, 底层使用char []又被final修饰, 所以不能被继承 
	- length() contains() indexOf() toCharArray() subString() trim() replace() equals() charAt()
- StringBuilder的效率最高, 在原有的字符串char数组进行修改, 而不是创建新的对象, 所以常用于字符串的拼接, StringBuffer每个方法都加上了synchronize关键字, 所以只有它是线程安全的
- String常用于字符串不发生改变场景

- Interger的装箱缓存机制: 对于 -128 到 127 的值会做一个缓存(类似乎常量池的小范围的缓存池), 使他们指向同一个对象
```java
// 可以在运行的时候添加 `-Djava.lang.Integer.IntegerCache.high=1000` 来调整缓存池的最大值。
Integer a = 128;  
Integer  
System.out.println(a == b); // false  比较的是对象
System.out.println(a.equals(b)); // true

new Integer(10) == new Integer(10) // false 都创建了新的堆内存对象 
```

- throwable是所有错误和异常的基类: 分error和exception两分支:
	- 编译时异常 error 在程序编译运行就不通过
	- 运行时异常 exception 在程序运行时的异常单 需要手动抛出异常和捕获处理异常 不会影响程执行
- finally块执行在return之前, 并且一定会被执行

- Java虽然有垃圾回收机制, 但是对于那些有着**大量元素的集合对象**, 如果不主动清理或者断开引用, GC无法及时回收, 就会造成内存泄漏问题, 因此对于那些占有大量内存的集合对象, 我们要使用集合对象的clear方法清空集合, 并进行null的赋值
