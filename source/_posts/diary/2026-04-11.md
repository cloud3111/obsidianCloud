# 2026-04-11
## Java八股复习
- 浅拷贝和深拷贝
	- 对于beanUtil的拷贝是属于浅拷贝, 两者其实指向的都是堆中的同一个对象, 也就是他们的地址值(指针)一样
-  comparable和comparator的区别
	- 首先comparable属于自然排序, 定义在类内部, 在类创建就定义好了, 而comparator属于定制排序, 只有在特殊情况下才会在new对象的时候去定义排序方式;
	- 其次comparable接口需要实现, 重写的是compareTo方法, 而comparator的重写compare方法则只需要在外部重写即可
- 注解的作用
	- 首先注解只起到一个标志的作用, 主要参数有target, retention等, 还可以在注解里面定义属性
	- 其次, 注解通常和反射一起被使用, 比如说@Aspect注解, 在容器启动阶段扫描带有切面注解的类或方法, 通过反射的getAnnotation方法获取注解信息, 然后执行对应逻辑
- 静态代理和动态代理
	- 静态代理是在编译时就生成了代理和被代理类
	- 动态代理是在运行时通过反射和字节码生成的
		- CGLIB代理是通过反射生成代理类, 创建成本高, 基于proxy.方法调用, 方法执行效率低
		- JDK代理是通过字节码生成代理子类, 创建成本低, 但是基于super关键字的方法执行效率高  
- 怎么实现拦截器? 和过滤器有什么区别
	- 实现handleInterceptor接口, 重写3个方法prehandle postHandle afterHandle
	- 首先, 过滤器由web容器管理, 在请求进入SpringMVC之前被执行, 用于做编码或者底层过滤; 而拦截器由SpringMVC提供, 在请求进入controller之前被执行, 用于做登陆校验, 权限管控
- hashMap的底层
	- 通过hashCode和equals保证对象的唯一性
	- jdk8之前采用链地址法将数组桶长度大于等于64, 链表长度大于8的链表转为红黑树, 使时间复杂度从O(n)降低到O(logn)
	- 为什么是8
		- 因为经过统计, 当小于8时链表性能会比红黑树好, 且红黑树节点占用更多内存
	- 为什么是64
		- 64 是扩容优化和树化成本之间的经验临界值，小容量先扩容，大容量再树化
- 什么是AQS
	- 是JUC包下实现很多的锁的底层框架, 采用volition关键字修饰的state状态变量表示同步状态, 保证可见性和有序性, 同时结合 CAS 保证修改 state 时的原子性
	- tryAcquire尝试获取锁，如果失败则进入 FIFO 双向队列等待；释放锁时通过 `tryRelease()` 释放并唤醒后继线程。
- 为什么HashTable比ConcurrentHashMap慢这么多
	- 因为HashTable底层用的是悲观锁; 而ConcurrentHashMap用的是CAS+synchronized对桶分别加锁, 实现高并发下更细粒度的线程安全
- 实现线程的3种方式
	- 继承Thread类; 实现Runable接口; 实现Callable接口
	- 第三种可以有返回值, 支持异步结果获取; 可以抛异常    将任务用FutureTask包裹
- ThreadLocal的内存泄漏问题
	- ThreadLocal 的内存泄漏主要是因为 ThreadLocalMap 的 key 是弱引用，而 value 是强引用。当 ThreadLocal 对象被回收后，key 会变成 null，但 value 仍被线程强引用，尤其在线程池场景下线程长期存活，容易导致内存泄漏
- 什么是lambda表达式?什么是stream流api
	- lambda表达式本质上是一个语法糖, 进行了一个显示的方法重写, 而函数式接口和那个方法引用, 就是一个隐式的方法重写
	- 而stream流api就是在lambda表达式和函数式接口的基础上进行值的转换或者其他操作, 比如我们可以进行数组的排序过滤, 打印, 转换, 最后再用容器接收
